# General

## Требования к проекту

* Веб-приложение
* Язык программирования: .NET

## Общие требования

### Поддерживаемые языки

На первом этапе система поддерживает три языка:

* Русский
* Английский
* Иврит

### Языковая пара

* Языковая пара не имеет направления
* Пары (ru-en) и (en-ru) считаются одинаковыми
* Пара должна храниться в нормализованном виде (например: сортировка по коду языка)

### Коллекции

Коллекция — это набор карточек, связанный с двумя языками.

* При создании коллекции пользователь выбирает два языка
* Коллекция создается пустой
* В коллекцию можно:

  * Добавить новую карточку
  * Присоединить существующую карточку с той же языковой парой
* Одна карточка может находиться в нескольких коллекциях
* Нельзя начать тренировку коллекции без карточек (кнопка неактивна)

### Карточка (TermCard)

Карточка содержит:

* Языковую пару
* Два значения (по одному на каждый язык)

Для каждого значения:

* Текст (максимальная длина 50 символов)
* Опциональная картинка
* Аудио (произношение)

### Аудио (произношение)

* Генерируется автоматически при создании карточки
* Генерация асинхронная
* Статусы:

  * Pending
  * Generated
  * Failed

### Ограничения текущей версии

* Нет дополнительных полей (часть речи, транскрипция и т.д.)
* Нет защиты от дублей
* Нет алгоритма интервального повторения

### Архитектурные решения

Карточки не принадлежат напрямую коллекциям.

Используется связь многие-ко-многим:

Collection ← CollectionCard → TermCard

### Дополнительные данные коллекции

Для каждой коллекции необходимо хранить:

* Количество карточек
* Дату создания
* Дату последнего изменения

---

## Главный экран

Таблица коллекций содержит:

* Название коллекции
* Количество карточек
* Дату создания
* Кнопку **"Добавить карточку"** — открывает попап создания карточки
* Кнопку **"Тренировка"** — неактивна, если в коллекции нет карточек

Пользователь попадает на экран тренировки при нажатии кнопки **"Тренировка"** в таблице коллекций.

---

## Экран тренировки коллекции

### Сессия тренировки (TrainingSession)

При начале тренировки создаётся сессия:

* Карточки перемешиваются случайным образом
* Формируется очередь карточек для прохождения
* Всего карточек = количество карточек в коллекции на момент старта
* Сессия существует до:

  * Ручного завершения пользователем
  * Или пока очередь карточек не станет пустой

---

### Завершение сессии

Сессия может завершиться двумя способами:

1. Ручное завершение:

* Пользователь нажимает кнопку "Завершить"

2. Автоматическое завершение:

* Очередь карточек пуста

---

### Элементы экрана

#### 1. Кнопка "Завершить"

* Завершает текущую сессию
* Возвращает пользователя на экран списка коллекций

---

#### 2. Выбор языка (ComboBox)

* Отображает текущий язык тренировки
* Пользователь может выбрать один из двух языков коллекции
* Выбранный язык определяет:

  * Отображаемый текст
  * Воспроизводимое аудио
  * Отображаемую картинку

⚠️ При изменении языка:

* Текущая сессия сбрасывается
* Начинается новая TrainingSession

---

#### 3. Прогресс

* Отображает статистику:

  * Правильно
  * Неправильно
  * Всего карточек
* Всего карточек фиксируется на старте сессии
* Формат: **"Правильно / Неправильно / Всего"**
* Пример: **"5 / 2 / 20"**

---

#### 4. Карточка

Карточка состоит из:

* Картинка (для выбранного языка)
* Текст
* Кнопка воспроизведения аудио

📌 При показе карточки:

* Аудио воспроизводится автоматически

📌 Если картинка отсутствует:

* Блок отображается пустым

---

#### 5. Кнопка "Не помню"

* Переключает карточку на второй язык

* После нажатия:

  * Меняется текст
  * Меняется картинка
  * Меняется аудио

* Карточка считается отвеченной с ошибкой

* Увеличивает счётчик "Неправильно"

* Кнопка "Не помню" блокируется

* Кнопка "Помню" блокируется

* Появляется кнопка "Далее"

* Карточка добавляется в конец очереди

---

#### 6. Кнопка "Помню"

* Активна, пока не нажата "Не помню"
* Переход к следующей карточке
* Карточка считается успешно пройденной
* Увеличивает счётчик "Правильно"

---

#### 7. Кнопка "Далее"

* Видна только после нажатия "Не помню"
* Переход к следующей карточке
* Не изменяет счётчики

---

#### 8. Кнопка "Добавить карточку"

* Открывает попап создания карточки (TermCard)
* Языковая пара определяется коллекцией

---

### Поведение очереди карточек

* В начале сессии порядок случайный
* При нажатии "Не помню":

  * карточка добавляется в конец очереди
* Сессия завершается, когда очередь пуста

---

## Экран "Новая карточка" (TermCard)

### Общая информация

* Экран реализован в виде попапа (modal)
* Используется для создания новой карточки (TermCard)

---

### Точки входа

Экран может быть открыт из:

1. Главного экрана

   * Кнопка "Добавить карточку" в строке коллекции

2. Экрана тренировки

   * Кнопка "Добавить карточку"

📌 В обоих случаях:

* языковая пара берётся из коллекции
* пользователь не выбирает языки вручную

---

### Элементы экрана

#### 1. Текстовые поля

* Два текстбокса (по одному на каждый язык)
* Поля обязательны
* Максимальная длина: 50 символов
* Поддержка направления текста:

  * LTR для английского/русского
  * RTL для иврита

---

#### 2. Картинки

Для каждого языка:

* Возможности:

  * Вставка через Ctrl+V
  * Upload файла
* Drag & Drop — не поддерживается
* Отображается превью
* Есть возможность удалить изображение
* Если изображение отсутствует:

  * отображается placeholder

---

#### 3. Аудио (произношение)

* Генерируется автоматически после ввода слова
* Генерация выполняется асинхронно
* Результат кэшируется
* Для каждого языка есть кнопка воспроизведения

Состояния:

* Loading (генерация)
* Ready (можно воспроизвести)
* Error (ошибка генерации)

---

### Кнопки действий

#### 1. "Сохранить"

* Создаёт карточку
* Закрывает попап
* Обновляет счётчик карточек в коллекции

---

#### 2. "Сохранить и продолжить"

* Создаёт карточку
* Очищает все поля
* Оставляет попап открытым

---

### Поведение формы

* При открытии:

  * фокус на первом текстовом поле
* После "Сохранить и продолжить":

  * форма очищается
  * фокус возвращается в первое поле

---

### Валидация

* Оба текстовых поля обязательны
* Максимальная длина: 50 символов
* Кнопки сохранения неактивны при невалидной форме
